# PoC 要件定義書 — Project Buzzing Map

## 目的
本ドキュメントは、Project Buzzing Map のコア機能「Video-to-Map」の技術的実現性を検証するための PoC 要件を定義する。
**本ドキュメントは OpenClaw（AIエージェント）への開発指示書として機能する。**

---

## Phase 2: Instagram スクレイピング

### 目的
Scrapling を使用して、Instagram から投稿データ（画像・動画）を取得できるかを検証する。

### 使用技術
- **Scrapling** (`StealthyFetcher` / `StealthySession`)
- Python 3.11+

### タスク 2-1: 画像投稿の取得
- **入力:** Instagram の公開アカウント URL または投稿 URL
- **処理:** Scrapling の `StealthyFetcher` を使い、投稿ページにアクセスして画像 URL を取得する
- **出力:** 画像 URL のリスト（JSON 形式）
- **成功条件:**
  - 少なくとも1件の公開投稿から画像 URL を取得できること
  - 取得した URL から実際に画像をダウンロードできること

### タスク 2-2: 動画投稿（Reel）の取得
- **入力:** Instagram の Reel URL
- **処理:** Scrapling で Reel ページにアクセスし、動画 URL を取得する
- **出力:** 動画 URL（直接ダウンロード可能な URL）
- **成功条件:**
  - 少なくとも1件の Reel から動画 URL を取得できること
  - 取得した URL から実際に動画ファイル（.mp4）をダウンロードできること

### タスク 2-3: アンチスクレイピング耐性テスト
- **入力:** 複数の Instagram ページ URL
- **処理:** 短期間に複数リクエストを送り、ブロックやCAPTCHAの発生状況を確認する
- **出力:** テスト結果レポート（成功率・エラー内容・推奨リクエスト間隔）
- **成功条件:**
  - `StealthyFetcher` 使用時にCloudflare等のボット検知を回避できること
  - 適切なリクエスト間隔の目安を特定できること

### 成果物
- `phase2-scraping/scrape_images.py` — 画像取得スクリプト
- `phase2-scraping/scrape_reels.py` — 動画取得スクリプト
- `phase2-scraping/anti_bot_test.py` — 耐性テストスクリプト
- `phase2-scraping/results/` — 取得結果サンプル（画像・動画・ログ）
- `phase2-scraping/REPORT.md` — 検証結果レポート

## Phase 3: 動画解析

### 目的
取得した動画から有用な情報（テロップのテキスト、音声の内容、画面の雰囲気）を抽出できるかを検証する。

### 使用技術
- **OCR:** Gemini Vision API（開発段階）
- **STT（音声→テキスト）:** Google Cloud Speech-to-Text または Whisper
- **映像分析:** Gemini Vision API（マルチモーダル）
- Python 3.11+, OpenCV（フレーム抽出用）

### タスク 3-1: テロップ抽出（OCR）
- **入力:** Phase 2 で取得した動画ファイル（.mp4）
- **処理:**
  1. OpenCV で動画からキーフレームを抽出（例: 1秒ごと、またはシーン変化検出）
  2. 各フレームを Gemini Vision API に送り、テロップ（テキスト）を抽出
- **出力:** フレームごとのテキスト抽出結果（JSON 形式、タイムスタンプ付き）
- **成功条件:**
  - 日本語テロップを 70% 以上の精度で読み取れること
  - 店舗名・メニュー名などの固有名詞を正しく認識できること

### タスク 3-2: 音声文字起こし（STT）
- **入力:** Phase 2 で取得した動画ファイルの音声トラック
- **処理:** 音声を抽出し、STT エンジンでテキスト化
- **出力:** 文字起こし結果（テキスト、タイムスタンプ付き）
- **成功条件:**
  - 日本語の音声を概ね正確にテキスト化できること
  - 店舗名・地名などの固有名詞が認識可能であること

### タスク 3-3: 雰囲気分析（Vision API）
- **入力:** Phase 2 で取得した動画のキーフレーム画像
- **処理:** Gemini Vision API にフレーム画像を送り、以下を分析させる:
  - 客層（年齢層、性別の傾向）
  - 環境（照明の明暗、賑やかさ、内装の雰囲気）
  - 料理（映っている料理の種類・見た目の特徴）
- **出力:** 分析結果（JSON 形式、各カテゴリのスコア/テキスト）
- **成功条件:**
  - 3つのカテゴリ（客層・環境・料理）について、人間が見ても妥当と感じる分析結果が得られること

### 成果物
- `phase3-video-analysis/extract_frames.py` — フレーム抽出スクリプト
- `phase3-video-analysis/ocr_telop.py` — テロップ OCR スクリプト
- `phase3-video-analysis/stt_audio.py` — 音声文字起こしスクリプト
- `phase3-video-analysis/vibe_analysis.py` — 雰囲気分析スクリプト
- `phase3-video-analysis/results/` — 分析結果サンプル
- `phase3-video-analysis/REPORT.md` — 検証結果レポート

## Phase 4: 店舗特定 → 地図マッピング

### 目的
Phase 2・3 で抽出したテキスト情報から店舗を特定し、地図上にプロットできるかを検証する。

### 使用技術
- **店舗検索:** Google Places API（Text Search / Place Details）
- **地図表示:** Leaflet.js または Google Maps JavaScript API
- Python 3.11+

### タスク 4-1: テキストからの店舗特定
- **入力:** Phase 3 で抽出したテロップ・音声テキスト
- **処理:**
  1. テキストから店舗名候補を LLM で抽出（例: 「渋谷のカフェ〇〇」→「カフェ〇〇」）
  2. Google Places API の Text Search で検索し、候補を取得
  3. エリア情報（動画のタイトル・ハッシュタグ等）を使って候補を絞り込み
- **出力:** 店舗情報リスト（店舗名、住所、座標、Place ID、マッチ信頼度）
- **成功条件:**
  - テロップに店舗名が含まれるケースで、正しい店舗を特定できること
  - 特定結果の信頼度を判定できること（曖昧な場合は「低信頼」とマーク）

### タスク 4-2: 地図へのプロット
- **入力:** タスク 4-1 で取得した座標データ
- **処理:** 簡易的な HTML + Leaflet.js で地図を生成し、店舗をピン表示する
- **出力:** インタラクティブな HTML マップファイル
- **成功条件:**
  - 特定した店舗が地図上の正しい位置にピン表示されること
  - ピンをクリックすると店舗情報（名前・住所・元の動画テキスト）がポップアップ表示されること

### 成果物
- `phase4-mapping/identify_shops.py` — 店舗特定スクリプト
- `phase4-mapping/generate_map.py` — 地図生成スクリプト
- `phase4-mapping/map.html` — 生成された地図サンプル
- `phase4-mapping/results/` — 特定結果データ
- `phase4-mapping/REPORT.md` — 検証結果レポート

## 全体の成功基準

PoC 全体として、以下が達成されれば本番開発に進む判断材料とする：

| # | 基準 | 重要度 |
|---|---|---|
| 1 | Instagram の公開投稿から画像・動画を取得できる | **必須** |
| 2 | 動画からテロップ（テキスト）を抽出できる | **必須** |
| 3 | 抽出テキストから店舗を特定し位置情報を取得できる | **必須** |
| 4 | 音声からの文字起こしが実用的な精度で可能 | 推奨 |
| 5 | 雰囲気分析が妥当な結果を返す | 推奨 |
| 6 | アンチボット対策を回避して安定的にスクレイピングできる | **必須** |
| 7 | OpenClaw が Phase 2〜4 を自律的に遂行できる | **必須** |
